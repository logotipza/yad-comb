Техническое задание: AI Маркетинг-Комбайн (Yandex Edition)
1. Общее описание системы
Система представляет собой веб-приложение для автоматизации маркетинга в экосистеме Яндекса (Директ, Метрика, Вордстат).
Ключевая особенность: Весь «мозг» работает на российских LLM (YandexGPT 5.1, GigaChat), а управление строится на принципе Human-in-the-Loop (ИИ предлагает — человек одобряет — система исполняет).

2. Архитектурный стек
Backend: Python (FastAPI) — асинхронность критически важна для работы с API и LLM.

Frontend: Next.js + Tailwind CSS + Shadcn/UI (чистый, современный интерфейс).

База данных: PostgreSQL (для хранения истории кампаний и очереди задач) + Redis (кеш).

LLM Gateway: Единый интерфейс-адаптер для переключения между YandexGPT, GigaChat и другими моделями без правки кода логики.

3. Модульная структура
Модуль 1: «Старт и Анализ» (URL -> Стратегия)
Логика:

Пользователь вводит URL сайта и подключает токены.

Система парсит сайт и передает контент в YandexGPT Pro 5.1.

Модель формулирует: УТП, сегменты аудитории и топ-10 базовых ключевых запросов.

Через Yandex Search API проводится поиск конкурентов по этим запросам.

Результат: Интерактивный отчет с маркетинговым планом.

Модуль 2: «Семантика и Вордстат»
Логика:

Берем базовые запросы из Модуля 1.

Используем Yandex Wordstat API v2 (метод GetTop) для сбора расширенного списка фраз.

LLM-фильтрация: Нейросеть анализирует список и автоматически вычищает «мусорные» интенты (информационные запросы, конкурентов и т.д.).

Система предлагает список ключевиков и минус-слов.

Human-in-the-Loop: Пользователь галочками отмечает, что берем в работу.

Модуль 3: «Генерация и Запуск»
Логика:

Для одобренных ключей YandexGPT генерирует заголовки (до 56 симв.) и тексты (до 81 симв.).

GigaChat (через функцию text2image) генерирует варианты визуалов для РСЯ.

Пользователь видит превью объявлений.

При нажатии «Принять» система через Direct API v5 (Ads.add) сама создает кампании и объявления в кабинете.

Модуль 4: «Оптимизатор» (Регулярные отчеты)
Логика:

Раз в сутки система тянет данные из Yandex Metrika API (конверсии, отказы) и Direct API (расходы).

Формируется отчет: ROI = (Доход - Расход) / Расход * 100%.

Агент-оптимизатор анализирует данные и выкатывает список «Действий»:

«Отключить фразу X (CPA слишком высокий)».

«Добавить минус-слово Y (много кликов без лидов)».

«Поднять ставку на фразу Z (высокая конверсия)».

Кнопка «Применить все»: Все подтвержденные правки улетают в Директ автоматически.

4. Инструкции для API (Технические детали)
Интеграция с Яндекс.Директ (API v5)
Использовать асинхронные POST-запросы к https://api.direct.yandex.com/json/v5/.
Обязательные заголовки:

HTTP
Authorization: Bearer <token>
Client-Login: <login>
Accept-Language: ru
Content-Type: application/json
